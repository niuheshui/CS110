## 三、代码分层结构（老师已经搭好框架，你只填核心逻辑）
整个文件系统拆成5层，底层已经写好，你实现上层4个核心层：
1. **磁盘块层 diskimg**：读磁盘扇区/块，老师写完了，不用你改；
2. **inode层 inode**：根据inode编号读取inode信息、找到文件对应磁盘块；
3. **文件层 file**：给inode号，读取这个文件的数据块；
4. **文件夹层 directory**：遍历文件夹，根据文件名找到对应的inode；
5. **路径解析层 pathname**：输入完整绝对路径（比如 `/a/b/c.txt`），一步步找到目标文件inode。

## 四、推荐实现顺序（按难度从低到高）
1. 先写inode两个核心函数：根据inode号读inode、通过inode算出第N个数据块在哪；
2. 再写文件读取函数，做完这两步 `-i` 测试就能通过；
3. 实现文件夹查找：输入文件夹inode+文件名，匹配目录项；
4. 最后实现路径解析，拆分多级目录递归查找，做完 `-p` 测试全过。

## 六、Unix v6老式文件系统关键知识点（文档补充说明）
### 1. 磁盘上三种核心数据结构（C结构体）
- `filsys`：超级块，存整个文件系统全局信息；
- `inode`：每个文件/文件夹对应一个inode，存大小、权限、磁盘块地址；
- `direntv6`：文件夹里的一条记录，存文件名+对应inode编号。

### 2. 70年代硬件限制带来的特殊设计（踩坑重点）
1. 当年内存、磁盘空间极小，大量用16位短整数；
2. 文件大小是**24位**，C语言没有对应类型，拆成两个变量`i_size0`（高8位）+`i_size1`（低16位），老师提供函数帮你合并成正常数字；
3. inode编号**从1开始**，0号不用，1号inode固定是根目录`/`，很容易出现下标错一位的bug。

### 3. inode里mode标记位（判断文件属性）
inode的16位mode数字每一位有特殊含义：
1. 最高位：标记这个inode是否被占用（有没有文件）；
2. bit12：标记是不是大文件（大文件要用一级/二级间接块存数据）；
3. bit13、14：区分普通文件 / 文件夹。
