# 通俗大白话讲解这份文档
这是斯坦福CS110系统编程课程的第三次作业，一共4道编程题，全是Linux进程、管道、进程间通信、调试追踪、多进程并行计算相关，截止2017年10月23号交，给12天时间写。下面分四个任务挨个说人话：

## 任务1：C语言写一个pipeline管道函数（简单小题）
### 要干啥
实现一个函数，模拟shell里 `cmd1 | cmd2` 管道效果：
1. 开两个子进程，第一个程序的标准输出直接接到第二个程序的标准输入；
2. 两个进程同时跑、并行执行，不会串行等第一个跑完再跑第二个（比如两个sleep 10，整体只花10秒，不是20秒）；
3. 把子进程PID存进数组，函数本身不用等待子进程结束；
4. 不用处理各种系统调用失败的报错，默认所有操作都成功。

### 补充
单独一个小练习，后面作业会用到管道思想，但这题本身不依赖其他题目代码。

## 任务2：C++写通用子进程创建工具subprocess（基础工具，后面两题都要用）
### 要干啥
写一个C++函数，用来创建子进程，支持四种组合：
1. 父进程要不要给子进程发输入（往子进程stdin写数据）；
2. 父进程要不要读取子进程的输出（拿子进程stdout的数据）。
四种组合：(发输入/不发输入)+(读输出/不读输出)都要正常工作。

### 细节要求
1. 创建进程、管道等系统调用一旦失败，要抛出C++异常；
2. 返回一个结构体，存子进程PID、父子通信的管道文件描述符；
3. 这是底层工具，后面第三题trace、第四题farm农场并行程序全都要调用它，必须先写对。

## 任务3：C++写trace工具，模仿gdb/strace追踪程序系统调用（中等大题）
### 工具作用
运行一个程序，打印这个程序执行过程中调用的所有内核系统调用（比如open打开文件、write写东西、read读数据、exit退出等），分两种打印模式：
1. **简洁模式 --simple**：只打印系统调用编号+返回值，纯数字，没有详细信息；
2. **完整模式（不带参数）**：打印完整函数名、传入参数、错误名称+错误描述（比如打开无效文件会标EBADF 错误）。

### 核心技术ptrace（Linux系统调试API）
程序分两个角色：
- tracer：我们写的trace程序（父进程，监控者）
- tracee：被跟踪的目标程序（子进程）
流程逻辑：
1. 子进程主动告诉内核“允许被父进程跟踪”，然后暂停自己；
2. 父进程用ptrace读取子进程CPU寄存器：系统调用编号存在ORIG_RAX，6个函数参数存在rdi/rsi/rdx等寄存器；
3. 子进程每次进入/退出系统调用都会暂停，父进程读取寄存器信息打印；
4. 如果参数是字符串（比如打开的文件名），父进程还要读取子进程内存把字符串读出来显示。

### 额外说明
第一次运行会解析Linux内核源码缓存系统调用信息，启动很慢，之后会缓存文件提速；提供官方参考程序trace_soln可以对照自己代码效果。

## 任务4：C++写farm并行质数分解农场（最难大题，多进程负载均衡）
### 背景
给了一个Python脚本factor.py，作用：输入数字，输出数字的质因数分解结果。
脚本自带特性：启动加`--self-halting`参数后，一等待输入就主动暂停自己，要收到SIGCONT信号才会继续读取数字计算。
单进程跑多个大数会串行计算，耗时叠加，机器多核性能浪费。

### farm程序要实现的功能
1. 自动读取电脑CPU核心数量，**每个核心启动一个factor.py工作进程**，并且绑定进程固定跑在对应CPU核心上；
2. 主线程不断读取标准输入的数字（一行一个数字）；
3. 维护空闲工人进程：
   - 工作进程算完一个数字会暂停，给主线程发SIGCHLD信号，主线程标记它为空闲；
   - 没空闲进程时主线程阻塞等待，不循环空耗CPU；
4. 拿到空闲工人后，把数字发给它，发SIGCONT唤醒进程计算；
5. 所有工人输出分解结果到控制台，不用保证输入输出顺序；
6. 输入读完后，等待所有工人完成计算，关闭管道、回收所有子进程，正常退出。

### 关键逻辑拆解
1. 信号处理：SIGCHLD处理函数，标记计算完成的进程为空闲；
2. 阻塞等待空闲进程：用sigsuspend，避免死循环轮询；
3. 资源清理：程序结束时给所有子进程发EOF，等待全部退出，恢复默认信号处理。

### 例子
8核机器同时输入4个超大数字，4个进程并行计算，总耗时只等于单个数字分解时间，而不是4倍时长；官方提供farm_soln参考程序。

# 整体总结
这份作业由浅入深四道Linux系统编程题：
1. 基础管道实现（C）
2. 通用子进程工具（C++，底层依赖）
3. ptrace系统调用追踪工具（调试工具，进阶）
4. 多核并行任务调度农场（多进程、信号、负载均衡，综合大项目）
核心考察：进程创建、管道IPC进程通信、ptrace调试、信号处理、多核CPU亲和绑定、多进程任务调度。
